# 종족 & 콘텐츠 설계

> 유닛 수치는 **초기 밸런스 기준값**(`🔶`)이며 `data/units/`에서 조정. 시스템 규칙은 [GAME_DESIGN.md](GAME_DESIGN.md) 참고.
> 능력치 약어: HP / ATK(공격력) / AS(공격속도,초당) / MS(이동속도,칸/초) / RNG(사거리,칸) / DEF(방어력) / C(코스트,코인) / T(생산 티어).
> 레이어: 🟫지상 / 🟦공중 / 🟧땅속.

---

## 1. 자연 (Nature) — 인해전술

**핵심 메커니즘**
- 싸고 빠르고 수 많은 유닛으로 물량전.
- **부화장(Hatchery)**: 코인으로 해금. 동일 종류 유닛 2기를 부화장에 넣고 **1턴 대기 → 복제체 1기 생성**(무료 3번째 사본). 물량 스노우볼.
- **겹침 배치(스택)** `발전 후`: 발전 방향 선택(7턴~) 이후 **한 칸에 최대 3유닛 겹쳐 배치** 가능(자연 전용). 겹친 자연 아군은 전투 중 서로 충돌을 무시해 뭉쳐 다님 → 화력 집중, 단 광역기에 취약. (규칙 상세: GAME_DESIGN.md §4–5)

## 2. 인간 (Human) — 발전

**핵심 메커니즘 — 시대 진화**
- 코인을 지불해 **시대(Era)** 를 올림. 시대가 오를 때마다 **모든 유닛이 진화**(이름·외형 변경 + 능력치 상승). 예: 선사시대 원시인 → 철기시대 병사.
- 시대 단계(초기안): **선사 → 철기 → 중세 → 근대 → 현대** (5단계) `🔶`
- 시대 진화 비용은 단계마다 상승. 진화 시 전 유닛 능력치에 단계별 배수 적용(예: 단계당 +15% HP/ATK 등) `🔶`.

**기본 유닛 7종 (지상 6 + 공중 1)** — 표는 **철기(2단계) 기준** 예시 수치 `🔶`. 시대별로 스케일.

## 3. 마계 (Demon) — 양보다 질

**핵심 메커니즘**
- 개체 수는 적지만 강한 유닛. **군단장**(6턴 발전 선택)이 빌드의 중심.
- **프리즘(Prism)**: 코인으로 해금. **유닛을 제물로 바쳐 군단장을 강화**(능력치/특수능력 단계 상승). → 군단 vs 군단장 강화 사이의 자원 긴장.
- (설계 레버) 기본 유닛은 타 종족보다 **코스트·스탯이 높은 편**, 출전 인원을 다 못 채워도 개체로 압도. `🔶`
